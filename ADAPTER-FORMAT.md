# 适配器格式

`Perspective Adapter`（视角适配器）的任务，是把一个由女娲蒸馏出来的人物 Skill，转换成 5-8 个可用于提纲盘问的“视角镜片”（`Perspective Lens`）。

本文档定义可复现规则。正式 `SKILL.md` 应引用本文档，不要在主技能文件里重复完整算法。

## 镜片结构

```md
## {镜片名称}

- 来源：{来自人物 Skill 的哪个心智模型、决策启发式、反模式、边界或表达规则}
- 意图：{这个镜片要强化什么，或防止什么}
- 触发场景：{提纲讨论中什么时候优先使用}
- 问题族：
  - 定义型：{问题}
  - 场景型：{问题}
  - 取舍型：{问题}
- 推荐答案规则：{如何提出建议答案，同时不假装已经被确认}
- 验证问题：{如何判断这一点是否清楚}
- 停止标准：{什么状态算足够清楚}
- 失败模式：{这个镜片要抓住哪类坏提纲表现}
- 评分：
  - relevance:
  - risk_reduction:
  - distinctiveness:
  - actionability:
  - total:
```

## 七步算法

1. 从选中的人物 Skill 中抽取候选认知单元。
2. 把每个认知单元转换成“提纲失败风险”。
3. 从失败风险生成镜片候选。
4. 对候选镜片聚类、去重。
5. 用下方评分锚点评分排序。
6. 实例化追问模板。
7. 生成验证问题和停止标准。

## 第 5 步评分

使用公式：

```text
score = relevance + risk_reduction + distinctiveness + actionability
```

### relevance：主题相关性

| 分数 | 判断锚点 |
| --- | --- |
| 3 | 直接命中用户主题或目标产物的核心失败风险。 |
| 2 | 与主题有关，但不是最核心的失败风险。 |
| 1 | 对提纲普遍有用，但不针对当前主题。 |
| 0 | 与当前主题或产物无关。 |

### risk_reduction：风险削减能力

| 分数 | 判断锚点 |
| --- | --- |
| 3 | 能防止核心判断变错、变虚或误导读者。 |
| 2 | 能防止重大结构问题，例如并列堆点、缺少取舍。 |
| 1 | 改善表达、风格或完整性，但不改变主逻辑。 |
| 0 | 没有明确对应的失败风险。 |

### distinctiveness：人物独特性

| 分数 | 判断锚点 |
| --- | --- |
| 2 | 强烈体现该人物独有的心智模型或决策风格。 |
| 1 | 和该人物有关，但也可能适用于许多其他人。 |
| 0 | 只是通用提纲建议，没有人物视角签名。 |

### actionability：可追问性

| 分数 | 判断锚点 |
| --- | --- |
| 2 | 能生成一个具体的一问一答，并能给出合理推荐答案。 |
| 1 | 能生成问题，但用户可能难以直接回答。 |
| 0 | 只能生成抽象批评或口号。 |

### 同分排序

候选镜片同分时，按以下顺序优先：

1. `risk_reduction` 更高。
2. `relevance` 更高。
3. `distinctiveness` 更高。
4. 能覆盖定义型、场景型、取舍型等不同问题类型。
5. 来源优先级更高：心智模型优先于表达风格。

## 质量门槛

镜片被使用前，必须全部通过：

- 能生成一个具体问题。
- 能给出推荐答案。
- 能定义什么叫“清楚”。
- 能抓住一个被命名的提纲失败风险。
- 能体现选中人物的独特视角。
- 不重复另一个更高分镜片。
- 不越过人物 Skill 的诚实边界。

## 重试与降级

镜片生成后立刻运行质量门槛检查。

- 第 1 次：生成 5-8 个镜片。
- 第 1 次重试：根据质量门槛失败原因修复不合格镜片。
- 第 2 次重试：降为最强的 3-5 个镜片，并重新评分。
- 仍失败：降级为临时草图（Sketch）模式。

最低可用输出：

- 正式适配器：优先 5 个有效镜片，最低 3 个有效镜片。
- 临时草图：最低 3 个粗粒度镜片。

如果少于 3 个镜片通过，停止适配器生成，并请用户换人物视角，或先做正式女娲蒸馏。

## 临时草图

仅在请求的人物 Skill 不存在，或适配器生成失败时使用临时草图（Sketch）。

临时草图规则：

- `status` 标记为 `sketch`。
- 只生成 3-5 个粗粒度镜片。
- 不得声称临时草图等同于正式女娲蒸馏视角。
- 最终输出必须提醒：该提纲应在正式蒸馏后复核。
- 临时草图不能直接提升为 `valid`；只有当真实人物 Skill 存在后，重新运行适配器，才可以把临时草图作为参考输入。

## 缓存元数据

适配器缓存放在 `adapters/{source_skill}.md`。

使用以下 frontmatter：

```yaml
---
source_skill: yuan-jiajun-perspective
source_skill_path: /absolute/or/relative/path/to/SKILL.md
source_skill_version: v1.2
source_skill_fingerprint: optional-hash-or-mtime
generated_at: 2026-05-29
adapter_algorithm_version: v1
status: valid
stale_reason:
---
```

`status` 允许值：

- `valid`
- `stale`
- `sketch`

## 缓存新鲜度

按以下顺序检测来源是否更新：

1. 如果来源 Skill 声明了版本号，比较 `source_skill_version`。
2. 如果没有版本号，比较 `source_skill_fingerprint`。
3. 如果两者都没有，比较 `generated_at`。
4. 如果以上信息都不可用，缓存可用于讨论，但在最终输出前必须提醒用户“未验证新鲜度”。

以下任一情况触发 `stale`：

- `source_skill_version` 和当前来源 Skill 版本不一致。
- `source_skill_fingerprint` 与当前来源文件不匹配。
- `generated_at` 距当前日期超过 90 天。
- 来源 Skill 文件已经不存在。

过期规则：

- 同一 major 版本，且最多落后一个 minor 版本：可用于讨论，但需要提醒用户。
- 落后超过一个 minor 版本：标记为 `stale`；只能用于阶段稿，或先重新生成。
- 主版本落后：不得用于最终输出；必须重新生成，或请用户接受临时草图级置信度。
- 来源 Skill 缺失：标记为 `stale`；提供正式蒸馏、临时草图、换现有视角三条路径。
- 仅因超过 90 天变为 `stale`：可用于讨论和阶段稿；最终输出前必须提醒用户该镜片可能需要重新生成。

## 缓存文件头

每个缓存适配器文件应以此开头：

```md
# 适配器：{source_skill}

状态：valid | stale | sketch
来源：{source_skill} {source_skill_version}
生成时间：{date}

## 视角镜片
...
```
